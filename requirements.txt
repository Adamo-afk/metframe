# Core ML / fine-tuning
#
# !!! ORDER MATTERS: install PyTorch with CUDA FIRST, then this file. !!!
#
# The default `pip install torch` (and any `pip install -r requirements.txt`
# that lists torch without an explicit index URL) fetches the CPU-only
# build. QLoRA training, HF inference, and BERTScore all allocate on GPU
# VRAM and will fail at runtime with:
#
#   ERROR: pipeline failed: CUDA is not available. QLoRA training
#   requires a GPU. Verify torch.cuda.is_available() returns True.
#
# Correct sequence (Windows / meteollm env):
#
#   1) pip uninstall -y torch torchvision torchaudio
#   2) pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cu126
#   3) python -c "import torch; assert torch.cuda.is_available(), 'CPU torch still installed'"
#   4) pip install -r requirements.txt
#
# Step 3 is the preflight: if the assert fails, step 4 will waste time
# installing dependencies around a torch that can't run the pipeline.
#
# torch is deliberately NOT listed as a package below, so this file never
# triggers a silent CPU-only reinstall. If a CUDA build is already
# present, pip leaves it alone.
# Upper bounds deliberate: transformers 5.x removed TrainingArguments.group_by_length
# and renamed Trainer(tokenizer=...) to Trainer(processing_class=...); trl 1.x moved
# DataCollatorForCompletionOnlyLM out of the public API; peft 0.17+ refactored
# prepare_model_for_kbit_training. Keep these ceilings until the training code is
# migrated to the new majors.
